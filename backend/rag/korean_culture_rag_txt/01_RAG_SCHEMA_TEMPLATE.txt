# RAG 카드 공통 스키마

문서 ID
- RAG 검색 및 Judge Node 로그에서 사용할 고유 ID입니다.

버전
- 문서 수정 이력을 관리하기 위한 버전입니다.

상위 카테고리
- indirect_intention, workplace_slang, implicit_reference, address_terms, tone, performance_pressure, responsibility, hierarchy 중 하나를 권장합니다.

하위 주제
- 실제 검색될 세부 상황입니다. 예: 지각 후 “지금 몇 시예요?” 질문형 질책.

핵심 설명
- 해당 표현이 어떤 문화적·상황적 의미를 가질 수 있는지 설명합니다.
- 절대 규칙으로 쓰지 말고 “가능성” 중심으로 작성합니다.

적용 조건
- 이 문서를 검색 결과로 사용해야 하는 조건입니다.
- 최소 2개 이상 충족될 때 강한 판단 근거로 사용합니다.

비적용 조건
- 이 문서를 적용하면 안 되거나, 낮은 confidence로만 써야 하는 조건입니다.
- 과잉 일반화를 막기 위한 필수 필드입니다.

판단 신호
- 상황, 관계, 어조, 직전 사건, 학습자 답변에서 확인할 신호입니다.

Judge Node 판정 기준
- pass: 학습자가 문화적 의도와 상황 목표를 이해하고 적절히 반응함.
- soft_pass: 의도는 일부 이해했으나 공손도, 자연스러움, 구체성이 부족함.
- fail: 표면 의미만 이해하거나 상황 목표와 반대되는 반응을 함.
- needs_clarification: 문맥, 관계, 어조 정보가 부족해 판단이 위험함.

대표 오해
- 외국인 학습자가 흔히 잘못 이해할 수 있는 지점입니다.

피드백 방향
- 학습자에게 설명할 핵심 메시지입니다.
- 비난하지 말고 “이 상황에서는 ~로 들릴 수 있다”처럼 안내합니다.

권장 대체 표현
- 학습자가 실제 상황에서 사용할 수 있는 자연스럽고 안전한 표현입니다.

예시
- scenario, ai_character_utterance, learner_response, expected_judgement, suggested_feedback을 포함합니다.

검색 키워드
- 한국어 키워드와 영어 보조 키워드를 함께 넣습니다.

중복 처리 규칙
- 비슷한 문서가 같이 검색될 때 어떤 문서를 우선해야 하는지 적습니다.
